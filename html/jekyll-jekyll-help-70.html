parkr Sunday, June 08, 2014

Does this mean that the main website is still the most authoritative source of documentation?

Yes yes yes! What the line you posted does is output the post URL, then prepend it with two metadata pieces: the baseurl (i.e. /hi in http://example.org/hi) and the site's URL, (i.e. http://example.org in http://example.org/hi). You can set these in the _config.yml file, but both are optional. They're there so that if your site requires a base url or the full URL, you don't have to change the templates, just the configuration file.
