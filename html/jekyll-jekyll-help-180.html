Template has page.excerpt, not post.excerpt.

boriseetgerink Thursday, October 30, 2014

In my head-template I tried to use the following HTML:

<meta name="description" content="{% if post.excerpt %}{{ post.excerpt | strip_html }}{% else %}{{ site.description }}{% endif %}">

This always displayed the site description, because post.excerpt is empty on a post. Changing it to the following code worked:

<meta name="description" content="{% if page.excerpt %}{{ page.excerpt | strip_html }}{% else %}{{ site.description }}{% endif %}">

The excerpt is stored on the page, not the post. Is that intended behavior? If so, I'll create a pull request to update the documentation.

I'm using Jekyll v2.4.0 on Windows 8.

troyswanson Sunday, November 02, 2014

page refers to the current page being displayed, post doesn't really refer to anything, unless you're using post as your iterating variable in a loop, like {% for post in site.posts %}.

I believe your change is correct; feel free to make a PR. Thanks! :smile:
