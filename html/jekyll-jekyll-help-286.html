Installing Jekyll is out-dated

starius Saturday, March 28, 2015

I'm trying to follow instructions to install Jekyll. I'm using ruby 1.9.1 I got the following error:

ERROR:  Error installing github-pages:
        public_suffix requires Ruby version >= 2.0.

This error happens if I run bundler install or gem install github-pages.

I think, it is a result of new version of public_suffix, released Mar. 25, 2015. I've found the following workaround: gem install public_suffix -v 1.4.6

After this, command gem install github-pages succeeded.

starius Monday, May 04, 2015

Hi @KurtPfeifle,

ERROR: Error installing github-pages: github-pages requires Ruby version >= 2.0.0.

This requirement changed in github-pages version 34, released April 8, 2015, after my comments were written (March 28, 2015 and April 3, 2015).

Try to set github-pages's version 33:

$ sudo gem install github-pages -v 33

By the way make sure that gem tool is also updated, when you update ruby. As I remember, they can be of different versions... I don't have Mac to check it though.
