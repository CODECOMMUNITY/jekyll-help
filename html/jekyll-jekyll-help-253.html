`:title` in collections seems to parse punctuation differently

beep Monday, January 26, 2015

I use a collection to manage “draft” entries, so I can print out a schedule of upcoming entries on my site. But when generating output files from that collection, it seems :title might be parsed differently in collections than it is elsewhere?

Here’s what I’m working from:

  • Post’s filename: _collection/2015-01-26-laphams-quarterly.markdown
  • Post’s title, in the front matter: `Lapham’s Quarterly”
  • Permalink definition for the collection: dir/:title:output_ext
  • Default permalink setting: dir/:title:output_ext

When served, this results in:

  • dir/lapham-s-quarterly.html if the post is in the collection; and
  • dir/laphams-quarterly.html if the post is in _posts.

(I am very likely doing something horribly wrong, but thought I’d log it here in case the issue’s relevant to anyone else.)

parkr Monday, January 26, 2015

Hey, @beep! Sorry you ran into this problemo. Turns out we extract the slug (used by :title) from the post filename, whereas collection documents' slugs are generated either from the title element in the YAML front matter or, if it doesn't exist, by the file basename.

It would make sense to me to call the slug the input file's basename in all cases (with date data stripped, of course). What say you?
