albertogg Wednesday, April 02, 2014

The gem --version will not necessarily be the same version as the package version (1.9.1). sudo update-alternatives --config gem should have done the symlink, check if /usr/bin/gem is symlinked or hardlinked to /usr/bin/gem1.9.1 or /etc/alternatives/gem. You can use this command ls -la /usr/bin | grep gem. If that's the case you are fine and can use the gem command normally.

rowoflo Wednesday, April 02, 2014

Yep, you are correct. I checked that gem is actually symlinked to /usr/bin/gem1.9.1. Just to make sure, I uninstalled Jekyll and reinstall with just the gem command everything still worked fine.
