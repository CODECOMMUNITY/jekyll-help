Best way to make an "about $author" partial

romariorios Tuesday, June 03, 2014

Hello.

I'm converting a blog to Jekyll and I want there to be an "About Author" box at the end of each post, containing the author's name, picture and description and I want this box to be consistent for each author. The easiest way to approach this is to provide the author's information in every posts written by them, but then there will be several copies of the same information across the articles written by that author.

I, then, figured I could create a plugin to create those partials, but, as far as I can see, Generator plugins are only able to create full pages, not just partials.

What, then, would be the best approach to do that?

budparr Tuesday, June 03, 2014

You can add authors to a _data/authors.yaml file and reference them in the front matter of each post. Might look like this: 

{% assign article_author = site.data.authors | where: 'author', 'author' %}
{% for author in article_author  %}
    {{author.name}}
    {{author.bio}}
{% endfor %}

edit: you'd probably need to capture the post's author beforehand with capture page.author. sorry, I left that out initially

budparr Tuesday, June 03, 2014

authors.yaml:

-
 name: Author Name
 email: email@gmail.com
 author_id: author_identifier

post.md

---
title: Post
layout: article
author: author_identifier
---
copy

template:


{% assign the_author = site.data.authors | where: 'author_id', {{page.author}} %}
{% for author in the_author %}
 name: {{author.name}}<br />
 email: {{author.email}}
{% endfor %}

I tend to use twitter as the author_id, and I tend to capture the page author before I use it in assign, but this will work.
