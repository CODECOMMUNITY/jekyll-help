Lots of white space -- will it affect performance?

tomjohnson1492 Tuesday, February 10, 2015

I have some code like this:

<li><a href="#">Scenarios</a>
<ul>
{% for page in site.pages | sort: "weight" %}
{% for tag in page.tags %}
{% if tag == "Scenarios" %}
<li><a class="post-link" href="{{ page.url | prepend: site.baseurl }}">{{ page.title }}</a></li>
{% endif %}
{% endfor %}
{% endfor %}
</ul>
</li>

When I view the source of the generated HTML, there's a TON of blank lines. Is that normal? Doesn't that affect performance?

alfredxing Tuesday, February 10, 2015

It is normal, since Liquid repeats newlines and whitespace in loops.

From what I know, the only downside to increased page size from whitespace is increased loading (downloading) time. From experience, this increase in time is extremely small (especially if gzip is enabled), and shouldn't be an issue at all.

budparr Tuesday, February 10, 2015

I think @penibelst has done some work on solving this issue: https://github.com/penibelst/jekyll-compress-html

penibelst Tuesday, February 10, 2015

As @budparr mentioned you can remove whitespaces and other optional things by using my Compress Layout.

@alfredxing The download performance affect is small, but performance is more than downloading: gzipping itself, rendering, and caching, especially on small devices, depend on the raw file size.
