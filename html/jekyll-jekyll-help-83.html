trans Monday, June 23, 2014

I don't understand how one is supposed to use YAML matter defaults. In the example given (reproduced below), the scope is listed as path: "about/blog" and type: "post". But all posts are stored in one place _posts. So is about/blog the destination somehow? But how would a blog post end up in about/blog when they almost certainly end up in :year/:month/:day/... ?

defaults:
  -
    scope:
      path: "" # empty string for all files
    values:
      layout: "my-site"
  -
    scope:
      path: "about/blog"
      type: "post"
    values:
      layout: "meta-blog" # overrides previous default layout
      author: "Dr. Hyde"
jaybe-jekyll Monday, June 23, 2014

Posts can be located in other locations, such as within subdirectories.

For example:

./about/blog/_posts/

Another, likely-more-appropriate example:

./technology/electronics/_posts/

Defaults specified with respect to the examples would apply to each example.
