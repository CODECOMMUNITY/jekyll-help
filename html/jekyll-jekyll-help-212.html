jaybe-jekyll Thursday, December 18, 2014

Glad you are enjoying Jekyll! It is indeed wonderful.

Regarding serving and auto-starting... Jekyll leverages Ruby' Webrick web server for local development serving. The Ruby' Webrick server should not be used for serving public/production. It is designed to provide a simple, local development server.

If your goal is to always have Jekyll running (--watch) and watching for changes to source files in order to re-render the [updated] site, that can be accomplished, i.e. jekyll serve -w.

For automatically starting unix processes at boot time, that is a system administration detail and task unrelated to Jekyll or any other server-side technology.

With that said, one could use cron, or init.d, or now, systemd, or the myriad other runtime management solutions.

Another popular way to dynamically build sites when they change is to use git along with git hooks that triggers jekyll to build and output to the served web directory. Note GitHub does this for you.

alfredxing Thursday, December 18, 2014

If you're planning to host the generated site for any prolonged period of time or to the public, I recommend serving the files with an actual web server like nginx or Apache. @jaybe-jekyll beat me to it, but I'll just reiterate that jekyll serve was meant to be used for development, not production.

For your hosting needs, I also recommend GitHub Pages, which can build Jekyll sites without plugins.
