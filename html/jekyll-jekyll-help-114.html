I think I figured it out after about 1000 tries - seems to work for all kinds of filters like category or other front matter variables - like "type" so I can have type: article or type: video and this seems to get tags from just one of them if I put that in the where part.

{% assign sorted_tags = site.tags | sort %}
{% for tag in sorted_tags %}
{% assign zz = tag[1] | where: "category", "Photoshop" | sort %}
{% if zz != empty %}

<li><span class="tag">{{ tag[0] }}</span>
<ul>
  {% for p in zz %}
  <li><a href="{{ p.url }}">{{ p.title }}</a></li>
  {% endfor %}
 </ul>
 </li>
 {% endif %}

 {% endfor %}

zz is just something to use to filter above the first tag[0] since all it seems to have is the tag itself, so you can't filter anything else with it. tag[1] has all of the other stuff.

Initially I was using if zz != null or if zz != "" but neither of them worked.
