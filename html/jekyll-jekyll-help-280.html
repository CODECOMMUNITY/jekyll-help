How can I increase site regeneration time given 50 capture tags?

tomjohnson1492 Wednesday, March 04, 2015

I have around 50 capture tags that I'm using to populate tooltips on another site using the method I described here: Create a help API.

I also want to embed the tooltips inside my jekyll site. I have a page that looks like this: 

{% for page in site.tooltips %}

{% if page.id == "sample1" %} {% capture sample1 %} {{page.content}} {% endcapture %}{% endif %}
{% if page.id == "sample2" %} {% capture sample2 %} {{page.content}} {% endcapture %}{% endif %}

// repeat this about 50 times with different page.id's...

{% endfor%}

When I implemented this code, my site's regeneration time jumped from 1.5 seconds to 11 seconds. How can I assign all these capture tags more efficiently?

tomjohnson1492 Friday, March 06, 2015

I realized that the for loops are requiring the long site build time. I moved everything into data files and now the site loads < 2 seconds.
