pootsbook Thursday, January 22, 2015

Great! If you're happy with everything working, fine, if not, read on:

Thanks for pasting the output, that's useful. The trouble installing nokogiri could be for one of two reasons:

  1. You don’t have the build toolchain installed
  2. You don’t have the package libxml2 installed (or you need to point to it correctly)

I would try installing the build tools via xcode-select --install on the command line, then re-running. If they are already installed, then there are a bunch of StackOverflow Questions about libxml2 and nokogiri but be careful as instructions vary depending on the environment (e.g. Mavericks vs Yosemite, Homebrew vs preinstalled).

To specify the path to libxml2 explicitly you can follow the instructions on the Nokogiri Install Page, especially the Using Nonstandard libxml2 / libxslt installations section. It involves installing nokogiri via gem install and passing in some configuration options. In terms of what the correct location is, I'm not sure, that's where my knowledge begins to end.

Of course it may be the build tools, that would be my first port of call; what's the result of running make -v on the command line? If it's similar toGNU Make 3.81 you have them. If not, run xcode-select --install.
