How can I allow people to read my Jekyll site offline?

tomjohnson1492 Monday, January 19, 2015

To avoid having to give offline readers a PDF, I want to simply deliver the _site files to readers. They would simply open the index.html file and load the content in a browser. However, right now it doesn't work. I have this in my _config.yml file:

baseurl: "/acme"

What would I need to change in the baseurl or elsewhere in order for people to be able to simply open index.html from _site without having jekyll installed and be able to view the site?

kleinfreund Monday, January 19, 2015

You need to make sure, that all your links (everything inside href="" and src="") are relative to the file. That means they must never start with a / or a protocol http://, etc. You need to reference ressources like this: src="../img/image.pnG". Than, if your site is static, it should work out of the box.

Great. Thanks!

kleinfreund Monday, January 19, 2015

Of course if another file is in the same directory, you just need href="about.html", etc.

The tradeoff with what I suggested above is maintainability. Writing out links relatively to files is a harder task than using the root (with /). However, the above works both online and offline. That's the whole point of it.
