Problem getting the "truncate" filter to work

budparr Saturday, June 14, 2014

jekyll (2.0.3), liquid (2.5.5)

I'm having a problem with the Liquid filter "truncate"

{% if item.front_matter_key %}
  {{ item.front_matter_key  | truncate: 1 }}
{% endif %}

For me, this returns nothing but ellipses ('...'), when if the front matter value was foo I'd expect the filter to return f

I've tried using capture and adding other filters to no avail. I'm assuming that anything on the Liquid for Designers page should work for us in Jekyll.

budparr Sunday, June 15, 2014

Figured this out. Liquid apparently, obviously, adds the ellipses as a result of the truncate filter, but what I didn't understand is that to get the desired result (of the value truncated to one character) you have to include the ellipses, so "truncate: 4" works, whereas whereas "truncate: 1" does not.

So, this seems strictly a liquid issue, which I will take up over there.
