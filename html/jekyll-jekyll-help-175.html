Final site problems

sarvagnan Wednesday, October 22, 2014

After running jekyll new in a directory, jekyll builds an example site. This site displays completely differently when using jekyll serve and when opening the html files on their own. This is a problem as the instructions say the the content of the _site folder is the final product but this does not appear to be so. When using jekyll serve, the site is properly formatted and styled. When opening the html files, there is no styling and the icons are very large. The CSS files have been generated though. Is there any way to fix this? Thanks in advance!

kleinfreund Wednesday, October 22, 2014

This is most likely an issue of how the CSS file is referenced from the default.html layout.

Inside _includes/head.html this line is causing you troubles: <link rel="stylesheet" href="{{ "/css/main.css" | prepend: site.baseurl }}">.

Without the first slash, it would also work locally, e.g. css/main.css. However if you set baseurl in your config, you end up with referencing site.tld/blogcss/main.css.

By the way, when I run jekyll new I get jekyll 2.4.0 | Error: You must specify a path.. Do I need to mkdir test && jekyll new test or something? Seems so.

That worked!! Thank you!!

You have to give jekyll new a location to run in. "jekyll new test" will create a folder called test and put all its stuff in there, so "mkdir test" isn't necessary.
