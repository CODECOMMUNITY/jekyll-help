willnorris Sunday, February 22, 2015

Regarding your first question... by default, jekyll expects to find your _plugins folder inside your source folder (alongside _layouts, _posts, etc).

skagedal Sunday, February 22, 2015

Allright, thanks! So that problem makes sense then.

willnorris Sunday, February 22, 2015

So, as alluded to in the error message, @site is nil when called here, which is ultimately what's throwing the error. This manual.rb plugin is really convoluted... it would probably be much better to set these up as standard jekyll pages and come up with a simpler approach to build the TOC (there are a number of TOC plugins out there)
