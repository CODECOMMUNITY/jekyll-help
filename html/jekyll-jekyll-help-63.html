RussEby Saturday, June 14, 2014

I noticed the same thing. I looked at the default site that Jekyll creates when you just type in new. The extra elements were there as well.

What I'm doing: I'm using the 'for page in site.pages' to create a li element for the nav section.

What it's doing: It's creating a li element for every file including the files without a page.title, thus causing an empty li element.

How I solved it: I did a test and it only made the li element if there was a page.title.

{% for page in site.pages %}
{% if page.title %}
<li>
 <a href="{{ page.url | prepend: site.baseurl }}">{{ page.title }}</a>
</li>
{% endif %}
{% endfor %}

Hope this helps.
