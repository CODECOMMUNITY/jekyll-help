Copy files verbatim to destination, without processing

clauswilke Saturday, December 20, 2014

There should be an option that tells Jekyll to copy files verbatim, without processing. At present, to my knowledge, there is no way that one can host a markdown file verbatim. This stackoverflow post suggests that one could host markdown files that don't have YAML front-matter, but that doesn't help when I want to host files that actually do contain such front-matter.

A simple configuration option like this: verbatim: [DIR, FILE, ...] modeled after exclude and include, would do the trick.

troyswanson Sunday, January 04, 2015

I haven't tested this, so I don't know if this would actually work, but try using a txt file extension instead of md or markdown. I believe Jekyll converts files based on their file extension, so if you give it a txt, it won't have a converter for it and just dump it as it sees it.

Let me know if that works, please.
