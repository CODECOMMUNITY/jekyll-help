troyswanson Thursday, June 26, 2014

@sleeke This looks like HTML that has been processed by a Markdown renderer only.

You need to include a layout value in the Front-matter that specifies which layout to use. Here's an example of how to set this up:

_layouts/post.html

<!DOCTYPE HTML>
<html>
<head>
  <title>Jekyll - {{ page.title }}</title>
  {% include head.html %}
</head>
<body>
  {{ content }}
</body>
</html>

_includes/head.html

  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1">

_posts/a-cool-guy.md

---
layout: post
---

# I'm a post!

This is a paragraph with some content.

There are three things in here that are worth knowing about, especially in the _layouts/post.html file

  • {{ page.title }} - This is a Liquid tag that will render the variable that was defined in the Front-matter. title is special; Jekyll will try to automatically populate this variable based on the name of the post file. You can override that by explicitly setting it in the Front-matter.
  • {% include head.html %} - This is another Liquid tag, but this one grabs the content from the file specified located in the _includes directory. This makes it easy to re-use HTML that should appear in multiple layouts.
  • {{ content }} - This is the rendered Markdown from your post. This will be the HTML that you gave in your original post.

Hopefully this helps! If you have any other questions, feel free to hit us up!
